# Wiki Schema & Conventions

This wiki follows Open Knowledge Format (OKF) v0.1 conventions.
See https://github.com/GoogleCloudPlatform/knowledge-catalog/blob/main/okf/SPEC.md

## Directory Structure

```
raw/                  Immutable source files (not indexed)
wiki/                 Wiki pages (free-form, no prescribed subdirectory layout)
  index.md            Content catalog (maintained by ingest agent)
  log.md              Chronological record (maintained by ingest agent)
WIKI_SCHEMA.md        This file
.gmd/config.cue       GMD configuration
```

## Naming Conventions

- Use kebab-case for page filenames: `transformer-architecture.md`
- Entity pages: singular noun (`kubernetes.md`, `attention-mechanism.md`)
- Concept pages: descriptive phrase (`scaled-dot-product-attention.md`)
- Comparison pages: `a-vs-b.md` format
- Source pages: `YYYY-MM-DD-short-title.md` format
- Synthesis pages: `YYYY-MM-DD-question-slug.md` format

## Page Format

Every wiki page must:
1. Start with YAML frontmatter (between `---` delimiters)
2. Have a single H1 heading matching the page title
3. Use standard markdown links for cross-references: `[Page Name](/path/to/page.md)`
4. End with a "See Also" section where applicable

## Frontmatter Schema

```yaml
---
type: concept          # REQUIRED by OKF: free-form string
title: Page Title      # OKF recommended: display title
description: Summary   # OKF recommended: auto-generated by summarizer
resource: https://...  # OKF recommended: canonical URI
tags: [tag1, tag2]     # OKF recommended: lowercase, kebab-case
timestamp: 2026-06-13T14:30:00Z  # OKF recommended: ISO 8601, set by tool on write
# GMD extensions:
status: draft          # Free-form: draft, reviewed, needs-update
sources: [page.md]     # Pages this content derives from
---
```

## The Index File (index.md)

The index is the wiki's table of contents. It is maintained by the ingest agent
and should not be manually edited. Format:

```markdown
# Wiki Index

## Entities
* [Entity Name](/wiki/entities/entity-name.md) — One-line description
* ...

## Concepts
* [Concept Name](/wiki/concepts/concept-name.md) — One-line description
* ...

## Comparisons
* [A vs B](/wiki/comparisons/a-vs-b.md) — One-line description
* ...

## Sources
* [Source Title](/wiki/sources/YYYY-MM-DD-title-slug.md) — Summary and date ingested
* ...

## Last Updated
YYYY-MM-DD HH:MM
```

## The Log File (log.md)

The log records every ingest operation chronologically. It is maintained by the
ingest agent and should not be manually edited. Format:

```markdown
# Wiki Log

## [2026-05-31 14:30] ingest | Source Title
- Created: entities/foo.md, concepts/bar.md
- Updated: entities/baz.md
- Flagged contradiction: claim X vs existing page Y

## [2026-05-30 10:15] query-save | "question text"
- Saved: wiki/synthesis/2026-05-30-question-slug.md
```

## Status Lifecycle

```
draft → reviewed → needs-update → reviewed → ...
```

- draft: Initial creation, needs review
- reviewed: Verified for accuracy
- needs-update: New information available, needs revision
